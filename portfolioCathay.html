Design System ⤵

Overview ⤵

Homepage

The primary goal of the homepage is to encourage users to take loan actions without overwhelming them with excessive details that might divert their attention.

Before
The color scheme was monotonous, lacking visual focus, which led to distractions.
Users had to navigate to another page to view terms and conditions, increasing unnecessary steps and reducing intuitiveness.
Charts were unclear and difficult to interpret.
After
The new color scheme is fresh and minimalistic, enhancing contrast between text and background for better readability.
Terms and conditions are now accessible via collapsible sections and upward scrolling, allowing users to interact seamlessly on the homepage.
Charts now follow a dashboard-like design, similar to a car speedometer, dynamically reflecting users’ loan and repayment behaviors. Increased contrast improves clarity and readability.

Loan Application Process

The loan application process involves extensive information such as regulations, terms, loan amount, interest rates, repayment periods, and account management fees. While some details are repetitive, they are necessary. However, excessive steps in the application process can cause user drop-off, making information integration crucial.

Before
Functions and information lacked hierarchy, making the interface cluttered and difficult to read, potentially causing users to abandon the process.
The previous loan amount input field misled users into thinking they could enter values below 10,000, and the shortcut buttons were rarely used.
The screen was fully occupied with information, requiring users to scroll down to discover additional input fields.
After
The interface now separates action areas and forms with a gray background, creating clear visual contrast and a cleaner layout for better readability.
Secondary information is now collapsible, reducing screen clutter while ensuring that users can still access additional details when needed.
The input field now visually differentiates non-editable values below 10,000, preventing user confusion. The shortcut buttons have been removed, freeing up more space for better usability.
